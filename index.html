Automatic Syllabification

Words are made up of syllables, which are sequences of phonemes that "go together" in the sense that they contain a vowel center followed and/or preceded by short consonant sounds. Natural languages have their own unique syllable patterns, and these patterns are related to the set of sounds available in the language. When working with language data, especially in audio, it can be very useful to break utterances down by syllable, because syllables represent a unit of sound that is below the word level but above the phoneme level, which is useful for identifying properties of sound like stress.

Syllabifying IPA transcribed text is fairly easy. Look at this link for example, an automatic syllabification algorithm by Penn University based on text representation of words. Despite being an important aspect of audio analysis, the availability of an application that can automatically syllabify speech is limited. There are some algorithms that apply spectral analysis on the input audio file to identify syllable boundaries, but these require advanced mathematics to use and are not very intuitive. The hardest part usually is to obtain an alignment for the audio file and the text, then obtaining the syllabification becomes a matter of applying the maximum onset principle . The following application aims to gather these ideas and provide a convenient application that can automatically syllabify text and audio. Currently, I want to support automatic syllabification for English, German, and Arabic, but would like to add more languages in the future!
